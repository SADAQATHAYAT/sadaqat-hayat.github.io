FINAL YEAR PROJECT

AI Image Similarity Finder

Problem: Searching large image collections using metadata or manual tagging doesn't scale and fails when labels are incomplete or subjective.

Approach: Used a CNN (ResNet-based) feature extractor to generate embeddings for each image. Indexed embeddings for fast similarity search using cosine similarity. Optimized inference latency for real-time queries.

Impact: Achieved 92% similarity accuracy on test queries. Scaled to 10,000+ images. Reduced average search time to ~1.2 seconds per query.

TensorFlow Keras ResNet50 Flask Docker

Work Experience

Building production ML systems with measurable business impact

Data Science Intern

DevEntia Tech, Islamabad
March - October 2025 | 8 months
Joined a small tech consultancy working with clients who had data but no ML infrastructure. My role was to turn exploratory notebooks into deployable systems that could run without me.

What I Built

Sales Forecasting System (Client: E-commerce)
  • Problem: Client's manual forecasting was causing inventory stockouts and overstocking, costing ~20% in lost revenue and storage
  • Constraint: Had to work with incomplete data (missing promotions metadata, inconsistent timestamps)
  • Approach: Built LSTM model with engineered lag features and rolling stats; benchmarked against ARIMA and Prophet
  • Outcome: 23% accuracy improvement over baseline, deployed via Flask API serving daily forecasts
  • Trade-off: Chose LSTM over Transformer despite lower theoretical ceiling because client's data volume didn't justify training cost
Churn Prediction Engine (Client: SaaS platform)
  • Problem: No systematic way to identify at-risk customers; retention team worked reactively
  • Constraint: Imbalanced dataset (12% churn rate), no clear feature set
  • Approach: EDA to find behavioral signals, engineered engagement features, trained XGBoost with SHAP for interpretability
  • Outcome: 78% recall on high-risk users, contributed to 15% churn reduction over 4 months
  • Trade-off: Optimized for recall over precision to maximize early warnings, even at cost of false positives
Sentiment Analysis Pipeline (Client: Customer support)
  • Problem: Support team drowning in 50K+ monthly feedback items with no prioritization
  • Constraint: Needed real-time processing, couldn't afford batch delays
  • Approach: Fine-tuned BERT, built FastAPI service with Kafka for streaming
  • Outcome: 87% F1-score, processing 10K reviews/hour, <1s latency
  • Trade-off: Used distilled BERT variant for speed despite 3% accuracy drop

What I Learned

  • Production ML is mostly about handling edge cases, missing data, and drift monitoring
  • Model explainability isn't optional when stakeholders control deployment decisions
  • Docker + CI/CD are not "nice-to-haves"—they're survival requirements
  • 90% accuracy means nothing if the system can't run Monday morning without you
